Management Scripts

Cross-platform scripts for setting up, running, and building the project from the repo root. Two versions are provided — manage.sh for Mac/Linux and manage.bat for Windows.

Commands

CommandWhat it does
setupInstalls Backend dependencies (cd Backend && npm install) then installs Frontend dependencies (npm install) from the repo root.
devStarts Backend and Frontend concurrently using npx concurrently. On Windows, also opens http://localhost:5173/ in the browser automatically after a 5-second delay.
buildRuns npm run build from the repo root to produce a production Frontend build.

Usage

Windows (.bat)
Mac / Linux (.sh)
# from the repo root .\manage.bat setup # install all dependencies .\manage.bat dev # start backend + frontend, opens browser .\manage.bat build # production build
# from the repo root — make executable first if needed chmod +x manage.sh ./manage.sh setup # install all dependencies ./manage.sh dev # start backend + frontend concurrently ./manage.sh build # production build

manage.bat (Windows)

Uses start to launch concurrently in a new window named "DevLynx". After starting dev mode it waits 5 seconds then opens the frontend at http://localhost:5173/ automatically.

Seed Scripts

Standalone Node.js scripts for populating and maintaining the database. Run from the Backend/ folder.

Seeds the database with ~170 books: ~50 classics with real Open Library cover URLs plus 120 auto-generated titles. Skips books that fail validation rather than aborting.

seed()Main async entry. Connects, iterates SEED_BOOKS, calls addBook for each, then disconnects.

Creates or updates two fixed accounts using findOneAndUpdate with upsert. Credentials are printed to the console on completion.
